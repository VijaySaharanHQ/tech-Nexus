# 🛠️ Developer Resources

Welcome to the **Developer Resources** hub at Tech Nexus!

This is the builder's toolkit. Whether you need a quick starter template for a new project, a reusable code snippet, or a guide to ace your next technical interview, you will find it here.

## 🗂️ Resource Directory

Access our toolkit by choosing a category below:

### 1. [Project Templates & Boilerplates](./templates/)
Don't reinvent the wheel. Start your projects faster with pre-configured setups.
* **Resources:** React starter kits, Express.js API boilerplates, Python Flask setups, and Docker-compose files.
* **Usage:** Clone, install dependencies, and start coding immediately.

### 2. [Source Codes & Snippets](./source-codes/)
A library of reusable, heavily commented code snippets for common problems.
* **Resources:** Authentication middleware, database connection scripts, custom CSS animations, and utility functions.
* **Usage:** Copy, paste, and adapt to your specific project needs.

### 3. [Career & Interview Prep](./career-prep/)
Everything you need to land your first (or next) job in tech.
* **Resources:** Resume templates, behavioral interview questions, system design guides, and negotiation tips.
* **Note:** For coding interview questions (like LeetCode style), please see the [DSA section in the Coding Academy](../../coding-academy/dsa/).

## 🤝 How to Contribute to Developer Resources

Developers love shortcuts! Help us build the ultimate toolkit:

1. **Keep Templates Updated:** Frameworks update constantly. If you submit a React or Next.js template, please note the version number in a local `README.md`.
2. **Document Your Snippets:** If you submit a source code snippet, include a brief explanation of what it does, what arguments it takes, and what it returns.
3. **Generalize:** Ensure any code you submit doesn't contain hardcoded personal data, API keys, or project-specific business logic.

Please read our master [Contributing Guidelines](../../CONTRIBUTING.md) before submitting a Pull Request!
