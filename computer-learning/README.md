# 🖥️ Computer Learning Center

Welcome to the **Computer Learning Center** at Tech Nexus!

Before building complex software, you need to understand the machines that run it. This hub is dedicated to teaching computer fundamentals, network architecture, and essential digital survival skills.

## 🗂️ Curriculum Structure

Choose a learning path below to build your foundational knowledge:

### 1. [Operating Systems & Networking](./os-networking/)
Understand the invisible software that makes hardware work and how computers talk to each other.
* **Topics:** Windows/Linux/macOS basics, file systems, IP addresses, DNS, routers, and the OSI model.
* **Resources:** Command-line (CLI) cheat sheets, visual networking diagrams, and OS setup guides.

### 2. [Productivity & Digital Tools](./productivity/)
Master the software that powers the modern workplace and academic environments.
* **Topics:** Microsoft Office Suite (Excel, Word, PowerPoint), Google Workspace, Notion, and general workflow optimization.
* **Resources:** Shortcut lists, formula cheat sheets, and productivity templates.

### 3. [Cybersecurity & Awareness](./cybersecurity/)
Learn how to protect yourself and your data in an increasingly digital world.
* **Topics:** Password management, phishing awareness, basic cryptography, and safe browsing habits.
* **Resources:** Security checklists, privacy guides, and threat-modeling basics.

## 🤝 How to Contribute to the Computer Learning Center

We welcome tutorials, guides, and tips that help demystify technology! 

1. **Use Visuals:** When writing tutorials for Productivity or OS topics, please include screenshots! Place your images in an `assets/` folder next to your markdown file.
2. **Keep it Accessible:** Avoid heavy jargon. Assume the reader is opening a terminal or an Excel spreadsheet for the very first time.
3. **Cybersecurity Rule:** We only accept defensive and educational security content ("White Hat"). Do not submit malicious scripts, exploits, or cracking tools.

Please read our master [Contributing Guidelines](../../CONTRIBUTING.md) before submitting a Pull Request!
